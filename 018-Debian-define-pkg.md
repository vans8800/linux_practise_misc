## 背景信息

<img width="426" height="249" alt="image" src="https://github.com/user-attachments/assets/06864340-67ec-40ce-a568-e7927dd00978" />

某个系统仓库中有多个linux内核包，需要用apt更新到特定的版本。


## 操作说明
---

### **1. 更新软件源（可选但推荐）**

在安装之前，建议先更新一下本地的软件包索引：

```bash
sudo apt update
```

### **2. 运行安装命令**

使用以下格式的命令来安装特定版本：

```bash
sudo apt install linux-image-loongson-3=4.19.190.8.14-lnd.3
```

**命令详解：**

- `sudo`: 获取管理员权限。
- `apt install`: 安装软件包的指令。
- `linux-image-loongson-3`: 这是图片中显示的软件包名称（去掉了后面的版本号和架构后缀）。
- `=4.19.190.8.14-lnd.3`: 等号后面紧跟你想要安装的具体版本号。

### **3. 若需要安装对应的头文件（可选）**

通常安装内核时，你可能也需要对应版本的头文件（headers），若仓库里有的话，命令类似：

```bash
sudo apt install linux-headers-loongson-3=4.19.190.8.14-lnd.3
```

### **4. 锁定版本（防止自动升级）**

若你安装这个特定版本是为了避免它被自动升级到最新版，你可能需要“锁定”这个版本（hold）：

```bash
sudo apt-mark hold linux-image-loongson-3
```

**注意：** 请确保你输入的版本号（`4.19.190.8.14-lnd.3`）与你在图片中看到的完全一致，包括所有的数字和连字符。

### **1. 类似** `package=version` **的 APT 高级语法**

除了你提到的指定版本安装，APT（以及底层的 `apt-get`）还支持多种通过特殊符号来控制软件包状态的语法。



这些语法非常强大，可以帮你批量处理依赖关系或锁定特定状态。

以下是常用的特殊语法汇总：

| 语法符号             | 作用描述                                           | 示例命令                                  |
| :------------------- | :------------------------------------------------- | :---------------------------------------- |
| **`pkg=version`**    | **指定版本**：安装或降级到特定版本。               | `sudo apt install nginx=1.18.0`           |
| **`pkg/目标发行版`** | **指定发行版**：从特定仓库（如 backports）安装。   | `sudo apt install vim/bullseye-backports` |
| **`pkg^`**           | **升级包**：将包升级到最高可用版本。               | `sudo apt install vim^`                   |
| **`pkg-`**           | **移除包**：卸载软件包（保留配置文件）。           | `sudo apt remove nginx-`                  |
| **`pkg+`**           | **安装包**：强制安装（即使有依赖问题尝试修复）。   | `sudo apt install nginx+`                 |
| **`pkg.`**           | **自动移除**：移除该软件包及其不再需要的依赖。     | `sudo apt remove nginx.`                  |
| **`pkg\*`**          | **通配符**：匹配所有符合名称模式的包（需加引号）。 | `sudo apt install "linux-image*"`         |

**实用技巧：**

- **查看可用版本**：在安装前，你可以用 `apt-cache policy 包名` 来查看仓库里到底有哪些版本供你选择，避免输错版本号。
- **锁定版本**：若你安装了一个特定版本并不希望它以后被自动更新，记得运行 `sudo apt-mark hold 包名`。

------

### **2. 如何彻底卸载某个软件包**

仅仅使用 `apt remove` 往往是不够的，因为它会保留配置文件。若你想把某个软件包（比如刚才安装的内核或头文件）连根拔起，包括它的配置文件和不再需要的依赖项，请按以下步骤操作：

#### **第一步：使用** `purge` **清除主程序及配置**

`purge` 命令等同于 `remove --purge`，它会删除软件本身以及 `/etc/` 下的所有配置文件。

```bash
sudo apt purge linux-headers-loongson-3=4.19.190.8.14-lnd.3
```

*(注：卸载时通常不需要指定版本号，除非你安装了多个版本且只想卸载其中一个)*

#### **第二步：使用** `autoremove` **清理依赖**

安装内核时，系统可能自动安装了一些依赖包。卸载主程序后，这些依赖包可能就变成了“孤儿”。

```bash
sudo apt autoremove
```

#### **第三步（进阶）：清理残留的配置文件**

有时候，即使使用了 `purge`，某些旧版本的包可能会在系统包列表中留下 `rc` 状态（即已删除但配置文件残留）。

可以用以下命令彻底扫尾：

```bash
# 列出所有只有配置文件残留的包
dpkg -l | grep ^rc

# 彻底清除这些残留配置
sudo dpkg -P $(dpkg -l | grep ^rc | awk '{print $2}')
```

#### **第四步（可选）：清理安装包缓存**

若你想释放 `/var/cache/apt/archives` 中下载的 `.deb` 安装包文件：

```bash
sudo apt clean
```

**总结：** 最标准的“彻底卸载”组合拳通常是：
`sudo apt purge <包名> && sudo apt autoremove`
